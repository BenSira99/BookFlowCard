

# 📚 BiblioCard — Architecture Complète de l'Application

## 🎯 Vision du Projet

> Application mobile React Native servant de **carte numérique de bibliothèque** pour les membres inscrits, communiquant en **réseau local** avec le poste de la bibliothécaire.

## 🏗️ Architecture Globale du Système

```
┌──────────────────────────────────────────────────────────────────┐
│                    RÉSEAU LOCAL DE LA BIBLIOTHÈQUE               │
│                                                                  │
│  ┌─────────────────────┐    WiFi Local    ┌──────────────────┐  │
│  │   📱 APP MOBILE     │◄──────────────►  │  🖥️ POSTE        │  │
│  │   React Native      │   REST API      │  BIBLIOTHÉCAIRE  │  │
│  │   (Membres)         │   + WebSocket   │  (Admin Web)     │  │
│  │                     │   mDNS/Bonjour  │                  │  │
│  └─────────────────────┘                  └────────┬─────────┘  │
│                                                     │            │
│                                              ┌──────┴─────────┐ │
│                                              │  🗄️ SQLite     │ │
│                                              │  Base Locale   │ │
│                                              └────────────────┘ │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │  🔐 Communication sécurisée : TLS mutuel sur réseau local  │ │
│  │  📡 Découverte automatique du serveur via mDNS             │ │
│  │  🔄 Synchronisation offline-first avec file d'attente      │ │
│  └─────────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────────┘
```

## 📦 Modules Fonctionnels Complets

### Module 1 — 🔐 Authentification & Onboarding

```
┌─────────────────────────────────────────────────────────┐
│  MODULE : AUTHENTIFICATION & ONBOARDING                 │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  DESCRIPTION :                                          │
│  Premier contact du membre avec l'app. La               │
│  bibliothécaire fournit un code d'activation unique     │
│  lors de l'inscription physique. Le membre entre ce     │
│  code + crée son PIN personnel à 6 chiffres.           │
│  Authentification biométrique proposée ensuite.         │
│                                                         │
│  FONCTIONNALITÉS :                                      │
│  • Écran d'onboarding avec 3 slides de présentation    │
│  • Saisie du code d'activation (fourni par biblio)     │
│  • Création d'un PIN sécurisé à 6 chiffres            │
│  • Activation biométrie (empreinte/Face ID)            │
│  • Connexion au réseau local automatique (mDNS)       │
│  • Écran de login avec PIN ou biométrie               │
│  • Verrouillage après 5 tentatives échouées           │
│  • Session avec token JWT (expiration 24h)            │
│                                                         │
│  ANIMATIONS PREMIUM :                                   │
│  ✦ Onboarding : Parallax scroll avec particules        │
│    teal flottantes et illustrations animées Lottie     │
│  ✦ Code activation : Chaque chiffre entré fait un      │
│    ripple effect teal lumineux, validation = explosion │
│    de particules dorées + haptic feedback              │
│  ✦ PIN : Clavier custom avec onde lumineuse au touch,  │
│    points qui pulsent avec glow teal                   │
│  ✦ Biométrie : Animation d'empreinte digitale avec    │
│    scan lumineux teal qui descend + pulse success     │
│  ✦ Transition login→home : La carte membre se         │
│    matérialise en 3D avec un flip élégant             │
│                                                         │
│  SÉCURITÉ :                                             │
│  🛡️ CWE-307 : Rate limiting sur tentatives de PIN     │
│  🛡️ CWE-798 : Aucun credential en dur dans le code   │
│  🛡️ CWE-312 : PIN hashé avec bcrypt avant stockage   │
│  🛡️ OWASP M1 : Stockage sécurisé via Keychain/       │
│     Keystore natif (pas AsyncStorage pour secrets)     │
│  🛡️ OWASP M5 : Communication TLS même en local       │
│  🛡️ CVE-prevention : Dépendances auth auditées       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Module 2 — 🪪 Carte Membre Numérique

```
┌─────────────────────────────────────────────────────────┐
│  MODULE : CARTE MEMBRE NUMÉRIQUE                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  DESCRIPTION :                                          │
│  Cœur de l'application. Affiche la carte de membre     │
│  sous forme d'une carte premium avec QR code           │
│  dynamique que la bibliothécaire scanne depuis son     │
│  poste pour identifier le membre instantanément.       │
│  Le QR code se régénère toutes les 30 secondes        │
│  pour empêcher les captures d'écran frauduleuses.      │
│                                                         │
│  FONCTIONNALITÉS :                                      │
│  • Carte visuelle premium (nom, photo, n° membre)     │
│  • QR code dynamique rotatif (TOTP-based, 30s)        │
│  • Statut de l'abonnement (actif/expiré/suspendu)     │
│  • Date d'expiration avec countdown                    │
│  • Niveau/type de membre (étudiant, adulte, senior)   │
│  • Mode plein écran pour scan facile                   │
│  • Luminosité auto maximale à l'ouverture du QR       │
│  • Compteur de livres actuellement empruntés          │
│  • Mode hors-ligne avec dernier QR valide en cache    │
│                                                         │
│  ANIMATIONS PREMIUM :                                   │
│  ✦ Carte : Effet glassmorphism avec reflet qui suit   │
│    l'inclinaison du téléphone (gyroscope) — comme     │
│    une carte holographique. Bords lumineux teal       │
│  ✦ QR Code : Timer circulaire autour du QR, quand    │
│    il se régénère = morphing fluide de l'ancien       │
│    vers le nouveau QR avec effet de dissolution       │
│  ✦ Statut : Badge animé pulsant vert si actif,       │
│    shimmer warning si bientôt expiré                  │
│  ✦ Photo : Bordure animée gradient teal qui tourne   │
│    lentement comme un anneau lumineux                 │
│  ✦ Ouverture : La carte descend du haut avec un      │
│    spring bounce premium + ombre portée animée        │
│  ✦ Plein écran : Expansion fluide avec le QR qui     │
│    zoome au centre + fond navy blur progressif        │
│                                                         │
│  SÉCURITÉ :                                             │
│  🛡️ CWE-330 : QR généré avec TOTP (secret partagé    │
│     + timestamp) — pas de valeur prévisible           │
│  🛡️ CWE-319 : QR payload chiffré AES-256            │
│  🛡️ OWASP M2 : Secret TOTP dans Keychain sécurisé   │
│  🛡️ OWASP M8 : Détection capture d'écran +          │
│     avertissement (le QR sera expiré de toute façon) │
│  🛡️ FLAG_SECURE : Empêche screenshot sur Android     │
│  🛡️ Anti-replay : Chaque QR contient un nonce unique │
│                                                         │
└─────────────────────────────────────────────────────────┘
```
